Presenters

Control how the server response updates the DOM

@inner — Replace innerHTML

The default presenter. Replaces the element's content with the response HTML.

▸ View code 
<button mx-click="@request:@inner" mx-path="/api/hello">
  Replace my content
</button>

@outer — Replace entire element

Replaces the element itself (not just its content). The button disappears and is replaced by the response.

▸ View code 
<button mx-click="@request:@outer" mx-path="/api/outer">
  Replace me entirely
</button>

@text — XSS-safe text rendering

Uses textContent instead of innerHTML. The server returns <script>alert(1)</script> — rendered safely as visible text.

@text (safe)
Click for safe text
@inner (unsafe — for comparison)
Click to inject HTML
▸ View code 
<!-- Safe: renders as visible text -->
<div mx-click="@request:@text" mx-path="/api/text">Click</div>

<!-- Unsafe: parses as HTML -->
<div mx-click="@request:@inner" mx-path="/api/text">Click</div>

@append — Append content

Adds new content after existing children. Perfect for chat feeds, logs, or activity streams.

▸ View code 
<button mx-click="@request:@append" mx-path="/api/items">
  + Add item
</button>
<div id="append-list"></div>

@prepend — Prepend content

Inserts new content before existing children. New items appear at the top — notification stack pattern.

▸ View code 
<button mx-click="@request:@prepend" mx-path="/api/items">
  + New notification
</button>

@id:target — Update by element ID

Updates a different element on the page, identified by its ID. The trigger and target are separate.

Waiting for update...
▸ View code 
<button mx-click="@request:@id:id-target" mx-path="/api/hello">
  Update target
</button>

<div id="id-target">Waiting...</div>

@class:name — Update all matching elements

Updates every element with the specified class simultaneously. One request, multiple targets.

Card A
Card B
Card C
▸ View code 
<button mx-click="@request:@class:broadcast-card"
        mx-path="/api/broadcast">
  Broadcast
</button>

<div class="broadcast-card">Card A</div>
<div class="broadcast-card">Card B</div>
<div class="broadcast-card">Card C</div>
