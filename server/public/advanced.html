@trigger / @dispatch — Event chaining

Dispatch a DOM event that other elements listen for. @trigger and @dispatch are aliases — both work identically.

A: Input → trigger form submit (debounced)

Form result appears here

B: Trigger event on target element (@dispatch alias)

Waiting for ping...
▸ View code 
<!-- Trigger form submit from input -->
<input mx-input.debounce.500ms="@trigger:submit" />

<!-- Dispatch event to another element -->
<button mx-click="@dispatch:ping:#ping-target">Ping</button>
<div id="ping-target" mx-ping="@request:@inner" ...></div>

Form Data Formats — @form:json @form:form @form:multipart

Serialize the nearest ancestor form in different formats. All three buttons share the same form.

Result appears here...
▸ View code 
<button mx-click="@request:@id:result"
        mx-method="POST" mx-path="/api/submit"
        mx-data="@form:json">
  Submit as JSON
</button>

Meta Tag Headers — mx-header

Headers defined in <meta mx-header> tags are automatically included in all requests. The server echoes them back.

<meta mx-header name="X-Demo-Header" content="mate.js-demo-server" />
Headers will appear here...
▸ View code 
<head>
  <meta mx-header name="Authorization" content="Bearer TOKEN" />
  <meta mx-header name="X-Demo-Header" content="value" />
</head>

Error Handling — mx-error event

Network failures dispatch an mx-error CustomEvent on the trigger element. The normal presentation flow is skipped.

Click a button to trigger an error
▸ View code 
<button mx-click="@request:@inner"
        mx-path="/api/error">
  Trigger error
</button>

<script>
  btn.addEventListener('mx-error', (e) => {
    console.log('Request failed:', e.detail);
  });
</script>

Action Aliases — @dispatch @passthrough

@dispatch is an alias for @trigger. @passthrough is an alias for @event. They behave identically — choose whichever reads better.

@trigger
@dispatch (same thing)
Click either button — same result
▸ View code 
<!-- These are identical -->
<button mx-click="@trigger:submit">Trigger</button>
<button mx-click="@dispatch:submit">Dispatch</button>

<!-- These are identical -->
<button mx-click="@event:@controller:method">Event</button>
<button mx-click="@passthrough:@controller:method">Passthrough</button>
