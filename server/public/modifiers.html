Event Modifiers

Chain modifiers with dots to control how events are handled

.prevent — Block default behavior

The link's navigation is blocked. The mate.js request fires instead.

This link won't navigate
Result appears here
▸ View code 
<a href="https://google.com"
   mx-click.prevent="@request:@id:prevent-result"
   mx-path="/api/hello">
  This link won't navigate
</a>

.stop — Stop propagation

Clicking the inner box only fires its handler. The outer box handler does NOT run.

Outer box (should NOT fire if you click inner)
Inner box (.stop — propagation blocked)
Outer: waiting
Inner: waiting
▸ View code 
<div mx-click="@request:@id:stop-outer" ...>
  Outer
  <div mx-click.stop="@request:@id:stop-inner" ...>
    Inner (.stop)
  </div>
</div>

.stopImmediate — Block all handlers on element

Two click handlers on the same element. The first uses .stopImmediate — the second never fires.

Handler 1: waiting
Handler 2: never fires
▸ View code 
<button
  mx-click.stopImmediate="@request:@id:stop-imm-first"
  mx-click="@request:@id:stop-imm-second">
  Click me
</button>

.once — Fire only once

The handler runs exactly once, then is removed. The button greys out after the first click.

▸ View code 
<button mx-click.once="@request:@inner" mx-path="/api/hello">
  Click me (works once)
</button>

.self — Only if clicking the element itself

Clicking the blue box fires the handler. Clicking a child element inside does NOT.

Click ME (the blue box)

← Clicking this child does NOT trigger
Waiting...
▸ View code 
<div mx-click.self="@request:@id:self-result" ...>
  <div>Child — clicking here does NOT trigger</div>
</div>

.outside — Click outside element

Triggers when you click anywhere except the designated element. Useful for closing dropdowns or modals.

Click ANYWHERE outside this box

Waiting for outside click...
▸ View code 
<div mx-click.outside="@request:@id:outside-result" ...>
  Click outside me
</div>

.debounce — Wait for pause

Fires only after typing stops for the specified duration. Compare three debounce timings side-by-side.

▸ View code 
<input mx-input.debounce.500ms="@request:@id:result"
       mx-path="/api/echo?key=term" />

.throttle — Limit fire rate

Mousemove fires hundreds of times per second. Throttle limits the handler to once per interval. Move your mouse in the box below.

Throttled (500ms)

Move mouse here

Result

Waiting...
▸ View code 
<div mx-mousemove.throttle.500ms="@request:@id:throttle-result"
     mx-path="/api/echo?key=throttled">
  Move mouse here
</div>

.capture + .passive

.capture fires during the capture phase (top-down) instead of bubble (bottom-up). .passive tells the browser the handler won't call preventDefault() for better scroll performance.

Parent (.capture — fires FIRST)
Child (normal — fires second)
Click a box to see order
▸ View code 
<!-- Capture: fires top-down -->
<div mx-click.capture="@request:@id:result" ...>
  <!-- Normal: fires bottom-up -->
  <div mx-click="@request:@id:result" ...>
    Child
  </div>
</div>
